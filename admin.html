📖 Talos Governance Guidelines

Document: Talos Governance Guidelines v1.0

This document defines the Tempered Convergence (TC) governance framework as implemented in the Billi BAI platform. It covers the principles, enforcement mechanisms, operator obligations, and escalation procedures that govern all AI engine changes. Updated as features and functionality evolve.

1. Tempered Convergence (TC) — the governing principle

Tempered Convergence is the AI governance framework developed as part of the Billi MPhil research (Stellenbosch USB, student 11603211). It governs the pace at which AI capabilities are deployed relative to the maturity of governance mechanisms. The core rule: no AI system in the Billi ecosystem acts autonomously on consequential decisions.

TC Rule 1 — Observe before acting
The PLM records observations for a minimum of 10 T+90 outcome cycles before any diagnostic signal is treated as actionable.
TC Rule 2 — Propose, don't deploy
The PLM generates proposals. It never deploys them. A human operator reviews every proposal before any engine parameter changes.
TC Rule 3 — Sandbox before authorise
Every proposal must pass a 7-archetype sandbox test with zero rank violations before the Authorise button is enabled.
TC Rule 4 — Log everything
Every authorisation or rejection is recorded in the immutable governance log with operator identity, timestamp, and rationale.
TC Rule 5 — REAI as the baseline
No proposal may be authorised that does not demonstrably improve at least one REAI component. If REAI drops below 60, all advisory outputs are suspended pending governance review.

2. What the PLM is and is not permitted to do

✅ PLM IS permitted to
Record observations on every /api/engine/rate call.
Run nightly diagnostic battery on the observation pool.
Generate parameter adjustment proposals (config delta objects).
Generate Novel Formula Unit (NFU) specifications.
Run sandbox tests against the 7-archetype battery.
Generate a new base64 RATING_ENGINE_CONFIG value after authorisation.
Surface proposals in the Talos Expert operator console.
🚫 PLM is NOT permitted to
Write to Railway environment variables directly.
Commit code to GitHub or any version control system.
Deploy any change without explicit operator authorisation.
Access user personal data (observations are pseudonymised).
Bypass the sandbox test gate under any circumstances.
Reduce the ADMIN_TOKEN requirement or weaken authentication.

3. The proposal lifecycle

Stage 1 — Proposed
PLM diagnostics generate a proposal. Operator is notified via the Proposals dashboard. Proposal status: proposed.
Stage 2 — Sandbox testing
Operator clicks "Run sandbox test". 7-archetype battery executes. All archetypes must maintain rank order (no ordinal violations). Status: sandbox_testing → pending_authorisation (if passed) or failed.
Stage 3 — Authorisation
Operator reviews the proposal and sandbox results. Clicks "Authorise" or "Reject". On authorisation: new RATING_ENGINE_CONFIG base64 value generated. Operator pastes it into Railway Variables manually.
Stage 4 — Deployment
Operator sets RATING_ENGINE_CONFIG in Railway. Service restarts. PLM records the config hash. REAI begins measuring improvement against the new baseline.
Stage 5 — Code changes (if required)
If a proposal requires changes to rating_engine.js (new ratio, new friction type, new Adizes variant), the PLM generates a human-readable specification — not code. The developer reviews, writes the change manually, and commits. The PLM never generates executable code for GitHub.

4. REAI — Rating Engine Accuracy Index

The REAI is a scalar metric (0–100) measuring how accurately the engine predicts real-world business outcomes. It is the PLM's performance gauge for the engine itself — the immune system's immune system.

Formula
REAI = 100 − (w₁·|MeanPredictionError| + w₂·CollisionMissRate×100 + w₃·RemediaryEfficacyGap + w₄·AdizesDisagreementRate×100)
Thresholds and responses
≥ 90: Engine performing well — no action needed.
80–89: Watch — diagnostic frequency increased.
70–79: Moderate drift — proposals expected.
60–69: Significant drift — mandatory operator review before new assessments presented as actionable.
< 60: Governance lockdown — PLM Mode 4 escalated immediately. All advisory outputs marked provisional.

5. Operator obligations

Proposal review SLA
Proposals that have passed sandbox testing should be reviewed within 14 days. Proposals that have been pending for more than 30 days without action are automatically flagged in the governance log.
Credential security
ADMIN_TOKEN must be a minimum of 64 hex characters (32 bytes, generated by crypto.randomBytes). It must never be shared, stored in Git, or transmitted outside Railway Variables. Rotate every 90 days or immediately on suspected compromise.
REAI monitoring
Operator checks the REAI dashboard at minimum weekly. Any REAI drop of more than 10 points in a 7-day period requires immediate investigation regardless of absolute score.

6. Escalation and incident response

AI output incident
Detection: REAI drop or user complaint.
Containment: add "provisional" disclaimer to all engine outputs (one line in app.html).
Correction: PLM proposal to address root cause.
Post-incident: root cause documented in governance log. User notification if DI was materially incorrect (>15 points from true value).
Governance lockdown (REAI < 60)
All DI scores marked "provisional — under review" in app.html. No new assessments saved to My Reports as authoritative. Operator must investigate and authorise a corrective proposal before normal operation resumes.

7. Third-party AI governance

Anthropic Claude API
Zero Data Retention (ZDR) must be formally confirmed before user financial data is processed at scale (WS-11.9). Until confirmed: all AI-processed content is treated as potentially retained. Anthropic model capability updates reviewed quarterly for governance impact.
Flowise orchestration
Flowise version pinned. Security review on every version update. No Flowise plugin installed without operator review. Flowise runs on Railway internal network only — not exposed directly to the internet.
